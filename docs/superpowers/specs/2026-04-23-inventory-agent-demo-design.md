# 库存管理 Agent — 移动端可交互 Demo 设计文档

> 日期：2026-04-23
> 状态：设计完成，待审阅

---

## 一、背景与目标

### 1.1 背景

扫描全能王平台约 4% 的扫描件为库存管理类单据（送货单 3w+、出库单 1w+、发货单/验收单/入库单各约 1w），用户以个人和小微商家为主。基于此场景洞察，计划构建库存管理 Agent，将扫描全能王从"拍照存档工具"升级为"AI 库存助手"。

### 1.2 Demo 目标

制作移动端可交互 HTML 原型，覆盖完整用户流程，用于：
- 内部团队/老板演示产品概念，争取立项资源
- 后续可用于用户访谈验证

### 1.3 Demo 不是什么

- 不接真实 OCR/LLM 服务，所有数据为 mock
- 不做后端，纯前端交互原型
- 不需要生产级代码质量，重点是流程完整和体验直观

---

## 二、已对齐的产品决策

| 决策点 | 结论 |
|--------|------|
| 产品形态 | 渐进式：先作为扫描全能王内嵌功能验证，跑通后考虑独立 |
| 目标市场 | 国内外同步推进（MVP 支持中文 + 英文） |
| MVP 范围 | 三个 Agent 能力构成最小闭环：单据语义理解 + 智能核对 + 智能检索 |
| 交互模式 | 混合模式：日常出入库走自动化（拍完即入库），复杂查询走对话式 |
| 数据存储 | 端云结合：默认本地，可选云同步 |
| 商业模式 | 纳入现有 VIP 会员体系 |
| 成功指标 | 主看功能渗透率（>30%），辅看用户留存 |

## 四、Demo 技术方案

### 4.1 技术选型

| 维度 | 方案 |
|------|------|
| 框架 | 单 HTML 文件 + CSS + JS，零依赖 |
| 移动端适配 | 固定 375×812 viewport（iPhone 尺寸），居中展示 |
| 页面路由 | JS 控制 div 显示/隐藏，模拟页面跳转动画（左右滑入） |
| 数据 | 内嵌 mock 数据（JSON 对象），覆盖中文单据场景 |
| 动画 | CSS transition 模拟：卡片滑入、loading 旋转、toast 弹出 |

### 4.2 Mock 数据设计

预置一套完整的中文示例数据：

- **供应商：** 王记五金、李氏钢材、华南电子
- **客户：** 张三建材、城东工地项目部
- **SKU 库：** 约 20 种商品（M8螺丝、M10螺母、扳手套装、电钻头、钢管、电线...）
- **历史单据：** 约 15-20 张（覆盖五种单据类型和各出入库场景）
- **库存台账：** 基于历史单据计算的当前库存

### 4.3 页面结构

```
index.html
├── 页面1: 扫描结果页（轻提示）
├── 页面2: 提取预览页
├── 页面3: 工作台首页
├── 页面4: 入库账本页
├── 页面5: 出库账本页
├── 页面6: 库存总账页
├── 页面7: 单据明细页（复用）
├── 页面8: 智能核对 - 选择单据
├── 页面9: 智能核对 - 结果页
├── 页面10: 智能搜索页
└── 公共组件: 底部导航栏、toast、loading、顶部导航栏
```

## 五、账本体系

### 5.1 入库账本

| 场景 | 说明 | 对应单据 |
|------|------|---------|
| 采购入库 | 供应商送货到仓，核验后入库 | 送货单、入库单 |
| 退货入库 | 客户退回商品，重新入库 | 退货单 |
| 调拨入库 | 从其他仓库/门店调入 | 调拨单 |
| 盘盈入库 | 盘点发现实物多于账面 | 盘点单 |

### 5.2 出库账本

| 场景 | 说明 | 对应单据 |
|------|------|---------|
| 销售出库 | 客户下单，拣货发货 | 出库单、发货单 |
| 退货出库 | 向供应商退货 | 退货单 |
| 调拨出库 | 调往其他仓库/门店 | 调拨单 |
| 报损出库 | 损坏/过期/丢失核销 | 报损单 |
| 领用出库 | 内部领用（工具、耗材） | 领用单 |

### 5.3 库存总账

按 SKU 汇总：当前库存 = 累计入库 - 累计出库，展示每个 SKU 的出入库明细时间线。

## 六、关键设计原则

1. **移动优先：** 所有交互按拇指操作区设计，主要操作在屏幕下半部
2. **零学习成本：** 流程自然衔接，不需要教程
3. **数据真实感：** mock 数据要像真实商家的单据，金额/数量/品名要合理
4. **动画到位：** 页面转场、卡片滑入、loading 等动画让 demo 有"产品感"而非"PPT 感"
5. **单文件交付：** 一个 HTML 文件，任何人收到后浏览器打开即可体验

## 七、成功标准

Demo 完成后应该能回答以下问题：
1. 用户扫完单据后，Agent 怎么介入？（轻提示流程）
2. Agent 能从单据中提取出什么？（结构化预览）
3. 日常怎么管库存？（工作台 + 入库/出库账本）
4. 对账怎么做？（智能核对）
5. 怎么查历史？（智能检索）
